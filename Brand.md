## 4. Visual Style: Aurora UI

BitcoinInfoNews adopts the **Aurora UI** design language — a system that merges the authority of financial media with the atmospheric depth of the crypto-native web.

### Five Pillars

**1. Dark-First Foundations**
Deep black and near-black surfaces form the canvas. This is a deliberate functional choice: reduces eye strain for readers monitoring markets at all hours, and maximizes contrast for data-heavy components.

**2. Luminous Accent Glows**
Bitcoin Amber (`#F7931A`) radiates with subtle bloom and radial gradient auras. Glow is used sparingly — only on hero elements, active states, and key CTAs — to preserve its impact.

**3. Glassmorphism Surfaces**
Cards, modals, and panels use frosted-glass layering: semi-transparent backgrounds with `backdrop-filter: blur()`, soft inner borders, and light diffusion. Glass depth creates visual hierarchy without weight.

**4. Sharp Typographic Hierarchy**
Headlines are bold and tight. Body text is spacious and legible. The type system creates an information architecture that guides the reader through complexity with minimal effort.

**5. Purposeful Motion**
Micro-animations on hover, transitions between states, and scroll-triggered reveals feel alive but never distract. Motion duration stays under 300ms. Nothing animates purely for decoration.

### Aurora UI Moodboard Reference
Visual references that define the aesthetic direction:
- Bloomberg Terminal meets Notion — structured information, dark surfaces
- Northern lights color palette — dark sky with luminous accent streaks
- Premium fintech apps (Robinhood dark mode, Phantom wallet, Ledger Live)
- Editorial rigor of The Athletic or The Information

*Aurora UI signals: modern, trustworthy, technically sophisticated, and premium — without being cold.*

## 5. Logo Usage

### Logo Anatomy
The **[B] InfoNews.** mark is composed of two elements:

- **Icon** — A geometric Bitcoin "B" symbol enclosed within square bracket connectors. The brackets evoke data structures, blockchain architecture, and editorial framing simultaneously.
- **Wordmark** — "InfoNews." set in a bold, geometric sans-serif. The terminal period (`.`) signals editorial finality and confidence — this is a statement, not a question.

### Color Combinations

| Variant | Icon | Wordmark | Background | Use case |
|---|---|---|---|---|
| **Primary** | Bitcoin Amber `#F7931A` | White `#FFFFFF` | Void Black `#0A0A0A` | Default: website, dark decks |
| **Reversed** | Bitcoin Amber `#F7931A` | Black `#0A0A0A` | White `#FFFFFF` | Light backgrounds only |
| **Monochrome Dark** | White `#FFFFFF` | White `#FFFFFF` | Any dark bg | Single-color print, dark emboss |
| **Monochrome Light** | Black `#0A0A0A` | Black `#0A0A0A` | Any light bg | Single-color print, light emboss |

### Logo Variations

| Variation | Description | When to use |
|---|---|---|
| **Full Lockup** | Icon + Wordmark horizontal | Primary use: header, footer, presentations |
| **Stacked** | Icon above Wordmark | Square formats: app icons, profile pictures |
| **Icon Only** | [B] symbol alone | Favicon, social avatar, app badge |
| **Wordmark Only** | "InfoNews." without icon | Co-branding contexts where icon clashes |

### Minimum Size

| Medium | Minimum Width |
|---|---|
| Digital (screen) | 120px (full lockup), 24px (icon only) |
| Print | 30mm (full lockup), 8mm (icon only) |

### Source File Formats

| Format | File Type | Use Case |
|---|---|---|
| **SVG** | `.svg` | Web, UI development, scalable digital use |
| **PNG (transparent)** | `.png` @2x, @3x | Digital presentations, social media, email |
| **EPS** | `.eps` | Print, merchandise, large-format production |
| **PDF** | `.pdf` | Print-ready press and agency delivery |

Always request the correct file type from the brand team. Never recreate the logo from screenshots or low-resolution exports.

### Clear Space
Maintain a minimum clear space equal to the full height of the [B] icon on all four sides of the logo in any usage.

### Logo Do Not

- Recolor the icon or wordmark outside approved palette
- Place on backgrounds that reduce contrast below WCAG AA ratio
- Add drop shadows, outlines, or glow effects to the logo itself
- Stretch, skew, or alter proportions in any axis
- Use the wordmark alone in primary brand contexts (always prefer icon + wordmark)
- Place any graphic element inside the logo's clear space
- Recreate or redraw the icon from scratch — always use approved source files

## 7. Typography

### Primary Typeface: Inter

**Inter** is the sole typeface for all UI and editorial content. It is open-source, optimized for screen rendering at all sizes, and carries the geometric precision that matches the Aurora UI aesthetic.

Font stack fallback: `'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', Helvetica, Arial, sans-serif`

### Secondary Typeface: JetBrains Mono

**JetBrains Mono** is used exclusively for data, prices, timestamps, and code. Its monospaced structure creates visual stability for numbers that change in real-time.

Font stack fallback: `'JetBrains Mono', 'Fira Code', 'Courier New', monospace`

### Type Scale

| Level | Size (Desktop) | Size (Mobile) | Weight | Line Height | Letter Spacing | Usage |
|---|---|---|---|---|---|---|
| **Display** | 64px | 40px | 800 | 1.05 | -0.03em | Hero headlines, homepage feature |
| **H1** | 48px | 32px | 700 | 1.1 | -0.02em | Article titles |
| **H2** | 32px | 24px | 700 | 1.2 | -0.01em | Section headers |
| **H3** | 24px | 20px | 600 | 1.3 | -0.01em | Subsection headers, card titles |
| **H4** | 18px | 18px | 600 | 1.4 | 0 | Widget headers, sidebar titles |
| **Body Large** | 18px | 17px | 400 | 1.75 | 0 | Feature article body copy |
| **Body** | 16px | 16px | 400 | 1.7 | 0 | Standard article body copy |
| **Body Small** | 14px | 14px | 400 | 1.6 | 0 | Summaries, pull quotes |
| **Label** | 13px | 13px | 500 | 1.5 | 0.02em | Tags, category labels, nav items |
| **Caption** | 12px | 12px | 400 | 1.5 | 0.01em | Author, date, image captions |
| **Data / Price** | 14–24px | 14–20px | 500 | 1.2 | 0 | Live prices, percentages (JetBrains Mono) |
| **Micro** | 11px | 11px | 500 | 1.4 | 0.04em | Badges, status pills, legal footnotes |

### Typography Rules

- Headlines always use sentence case — never ALL CAPS or Title Case Unless It Is a Proper Noun
- Body copy never exceeds 720px column width for readability
- Minimum body font size: 16px on all screens
- Never use more than two typefaces in a single layout
- Avoid orphaned words (single word on last line) in headlines — adjust manually
- Limit heading levels to H1–H3 within article content (H4+ for UI only)

*Guiding principle: Typography should feel like reading a premium financial terminal — not a blog.*

**Phosphor Icons** (Regular weight) is the icon library for all UI elements. Use the **Regular** (outline) style exclusively. The **Bold** weight may be used only for emphasis in empty states or hero illustrations.

## 10. Photography & Image Style

### Thumbnail Philosophy
Thumbnails are the first impression of every article. They must communicate the story's subject matter instantly, even at small sizes. BitcoinInfoNews thumbnails follow a consistent visual grammar — not random stock photography.

### Preferred Image Style

| Type | Description | Priority |
|---|---|---|
| **Data Visualization** | Custom charts, graphs, and infographics relevant to the story | Highest |
| **Branded Illustration** | Flat-style crypto illustrations on dark backgrounds (Bitcoin logos, blockchain graphics) | High |
| **Concept Photography** | Abstract, dark-toned photography — circuit boards, gold, finance architecture | Medium |
| **Event Photography** | Conference photos, exec portraits — for people-centric stories only | Low |

### Thumbnail Specifications

| Property | Value |
|---|---|
| **Aspect Ratio** | 16:9 (primary), 3:2 (fallback) |
| **Minimum Dimensions** | 1200 × 675px |
| **Recommended** | 1600 × 900px |
| **File Format** | WebP (primary), JPEG (fallback) |
| **Max File Size** | 200KB (WebP), 400KB (JPEG) |
| **Color Treatment** | Dark-dominant, desaturate stock photos by 10–20%, add subtle dark overlay |

### Thumbnail Text Rules
- Headline text on thumbnail: **Not permitted** (text belongs in the article card, not baked into the image)
- Watermarks: BitcoinInfoNews watermark optional on original illustrations only (bottom-right, 15% opacity)
- Logo on thumbnail: Never — logo appears in the card UI, not the image

### Image Overlay System
When using photography on dark surfaces, apply a gradient overlay:
- Top: `rgba(10,10,10,0)` → Bottom: `rgba(10,10,10,0.85)`
- Ensures text legibility on featured card format

### Image Caption Style

Captions appear directly below images in article body and must follow this treatment:

| Property | Value |
|---|---|
| **Font** | Inter |
| **Size** | 13px |
| **Weight** | 400 (regular) |
| **Style** | Italic |
| **Color** | Mist `#A0A0A0` |
| **Line height** | 1.5 |
| **Top margin** | 8px from image bottom |
| **Bottom margin** | 24px before next paragraph |
| **Max width** | Same as image width |

**Source attribution format:**
`Caption text. (Source: Organization Name / Photographer Name)`

Example: *Bitcoin mining facility in Texas. (Source: Luxor Technology)*

Never omit source attribution for images obtained from third parties. For original BitcoinInfoNews graphics, no attribution required.

### Image Do Not
- Use bright, colorful, saturated stock photos that break the dark aesthetic
- Use AI-generated faces or recognizable real people in illustrative thumbnails
- Use meme images or low-resolution graphics
- Reuse the same stock photo across multiple articles

## 12. Tone of Voice

### Core Personality
**Authoritative. Clear. Grounded. Never hype.**

BitcoinInfoNews speaks like a knowledgeable colleague in the Bitcoin space — someone who has done the reading, understands the nuance, and can explain it without showing off.

### Voice Attributes

| Attribute | What it means in practice |
|---|---|
| **Direct** | Lead with the point. No filler sentences. No clickbait framing. |
| **Precise** | Use accurate terminology. Avoid vague superlatives like "massive", "huge", "skyrocket". |
| **Calm under volatility** | When the market moves dramatically, the tone stays measured — not panicked, not euphoric. |
| **Educational without condescension** | Explain concepts as if the reader is intelligent but new to this specific topic. |
| **Transparent** | Clearly signal what is established fact, what is analysis, and what is uncertain. |

### Tone by Content Type

| Content Type | Tone | Style |
|---|---|---|
| **Breaking News** | Urgent but measured | Short sentences, inverted pyramid, facts first |
| **Analysis** | Confident, evidence-based | Can use first-person plural ("data suggests"), cite charts |
| **Guides** | Helpful, patient | Step-by-step, define terms on first use, use examples |
| **Market Updates** | Clinical, data-led | Numbers before opinions, percentage change before narrative |
| **Press Release** | Neutral, clearly attributed | Always attributed to the issuing company, never BitcoinInfoNews voice |

### Tone by Channel

| Channel | Tone | Notes |
|---|---|---|
| **Article body** | Authoritative, in-depth | Full sentences, complete sourcing |
| **Headline** | Direct, factual | Present tense, active voice, no puns |
| **Twitter/X** | Sharp, one-idea-per-tweet | Link to article, avoid opinion without sourcing |
| **Newsletter** | Slightly warmer | Can use brief personal framing, curated selection |
| **Push notification** | Ultra-concise | Max 60 chars, factual, no punctuation theatrics |

### Tone Examples

| ❌ Avoid | ✅ Prefer |
|---|---|
| "Bitcoin EXPLODES to new ATH!!" | "Bitcoin reaches a new all-time high as ETF inflows hit $1.2B in a single day" |
| "You need to buy this NOW before it's too late" | "Here is what the on-chain data shows and what analysts are saying" |
| "Crypto insiders reveal secret that banks don't want you to know" | "According to filings reviewed by BitcoinInfoNews, the policy would..." |
| "This altcoin is about to 100x" | Out of scope — Bitcoin-first editorial policy applies |
| "Some say Bitcoin could go up or down" | "Glassnode data shows long-term holders are at their lowest sell rate since 2021" |

## 13. Editorial Style Guide

This section governs all written content published under the BitcoinInfoNews masthead.

### Bitcoin Capitalization

| Usage | Correct | Incorrect |
|---|---|---|
| The network / protocol | Bitcoin | bitcoin |
| The currency / unit | bitcoin (lowercase) or BTC | Bitcoin (for the unit) |
| In headlines | Bitcoin (always cap in headlines) | — |
| Satoshi (unit) | satoshi / sats | Satoshi (unless referring to the person) |

> Rule of thumb: **Bitcoin** (capital B) = the network. **bitcoin** (lower b) = the money.

### Number Formatting

| Context | Format | Example |
|---|---|---|
| Prices under $1,000 | 2 decimal places | $94,250.32 |
| Prices over $1,000 | Comma separator, 2 decimal | $94,250.32 |
| Large round numbers | Abbreviate above $1M | $1.2B, $450M, $3.5T |
| BTC amounts | Up to 8 decimal places | 0.00042300 BTC |
| Percentages | 2 decimal places | +4.72%, -1.20% |
| Hashrate | Standard SI units | 850 EH/s |
| Volume | Always USD with symbol | $2.4B daily volume |

### Date & Time Formatting

| Context | Format | Example |
|---|---|---|
| Article date | Month DD, YYYY | June 1, 2026 |
| Timestamps | 24h UTC | 14:32 UTC |
| Relative time (fresh) | Human-readable | 3 hours ago |
| Historical dates | Month YYYY | March 2024 |

### Headline Rules

- **Tense:** Present tense for recent events ("Bitcoin Hits $100K"), past tense for concluded events ("Fed Raised Rates by 25bps")
- **Length:** 60–80 characters ideal; never exceed 110 characters
- **Active voice:** "SEC Approves Bitcoin ETF" not "Bitcoin ETF Approved by SEC"
- **No puns or wordplay** in hard news headlines (acceptable in opinion or feature only)
- **No all-caps** words in headlines except proper acronyms (ETF, SEC, BTC)
- **No question headlines** ("Will Bitcoin Reach $200K?") for hard news — use for analysis only

### Common Terms Reference

| Term | Correct | Never |
|---|---|---|
| Cryptocurrency | cryptocurrency (lowercase) | Cryptocurrency (unless start of sentence) |
| Blockchain | blockchain | BlockChain, block chain |
| Decentralized finance | DeFi | defi, Defi |
| Non-fungible token | NFT | nft, N.F.T. |
| All-time high | ATH | all time high, All-Time-High |
| Exchange-traded fund | ETF (spell out on first use) | E.T.F. |
| Proof of Work | Proof of Work / PoW | Proof-of-Work (no hyphens in text) |

### Source Attribution Standards

- All factual claims require a linked source or named attribution
- Acceptable: "According to Glassnode data...", "A CoinDesk report states...", "In an X post, [Name] wrote..."
- Unacceptable: "Sources say...", "It is reported that...", "Many believe..."
- Price data must cite the source exchange or aggregator: "Bitcoin traded at $94,250 on Coinbase at 14:00 UTC"

### Disclaimer Boilerplate

**Financial disclaimer** (append to all price analysis and market content):
> *"This content is for informational purposes only and does not constitute financial advice. Always conduct your own research before making investment decisions."*

**Press release disclaimer** (prepend to all PR content):
> *"The following is a press release from [Company Name]. BitcoinInfoNews is not responsible for the content and has not independently verified the claims made herein."*

**Sponsored content disclaimer** (prepend to all sponsored articles):
> *"This is sponsored content. BitcoinInfoNews received compensation for its publication. The views expressed are those of the sponsor and do not represent the editorial position of BitcoinInfoNews."*

## 15. Design Principles

### 1. Bitcoin-First Clarity
Every design decision starts with the question: *does this make Bitcoin information easier to find, understand, and trust?* Decoration serves communication — never the reverse. If a visual element does not aid comprehension, remove it.

### 2. Signal Over Noise
The interface must feel calm and structured even when covering market turbulence. Reduce visual clutter at every decision point. Use whitespace intentionally. Let headlines breathe. Let data speak without decoration around it.

### 3. Dark Canvas, Bright Signal
The dark background is not aesthetic — it is functional. Readers monitor markets at all hours. Dark surfaces reduce eye strain during extended sessions and create maximum contrast for the price data and charts that are central to the product experience.

### 4. Aurora Depth
Surfaces are layered, not flat. Glassmorphism cards, ambient glow, and subtle gradients create hierarchy and premium feel without adding visual weight. Depth should be felt, not seen.

### 5. Trust Through Transparency
Design must enforce editorial integrity. Editorial content and commercial content must be immediately distinguishable through visual treatment — color accents, labels, and layout cues. Readers must never have to wonder if a story was paid for.

### 6. Data as Design
Live prices, charts, heatmaps, and calculators are first-class design elements — not afterthoughts embedded from third parties. Data components must be beautiful, legible, and fast. A price widget is as much a part of the brand as the logo.

### 7. Responsive by Default
A majority of readers access BitcoinInfoNews on mobile while tracking markets or consuming quick updates. Every component is designed mobile-first. Touch targets minimum 44×44px. No hover-only interactions.

### 8. Speed is a Feature
Page performance is a design decision. Image optimization, lazy loading, minimal JavaScript, and efficient CSS are brand requirements — not engineering preferences. Slow pages lose readers.

## 16. Accessibility

BitcoinInfoNews is committed to WCAG 2.1 Level AA compliance across all digital products.

### Core Requirements

| Requirement | Standard | Implementation |
|---|---|---|
| **Color contrast — body text** | 4.5:1 minimum | White on Carbon passes at 15.8:1 ✅ |
| **Color contrast — large text** | 3:1 minimum | Bitcoin Amber on Void at 4.6:1 ✅ |
| **Focus indicators** | Visible on all interactive elements | 3px Bitcoin Amber outline, 2px offset |
| **Keyboard navigation** | Full site navigable without mouse | Tab order follows DOM order |
| **Images** | Alt text required on all images | Descriptive, not "image of..." |
| **Icons** | Label required alongside icons | `aria-label` on icon-only buttons |
| **Color as sole indicator** | Never use color alone to convey meaning | Price change uses + / – prefix plus color |
| **Motion** | Respect `prefers-reduced-motion` | All animations wrapped in media query |
| **Font size** | Minimum 16px body text | Never go below 11px for any visible text |
| **Touch targets** | Minimum 44×44px | All buttons and interactive elements |

### Semantic HTML Requirements
- One `<h1>` per page (article title or page title)
- Heading hierarchy must not skip levels (h1 → h2 → h3)
- All form inputs must have associated `<label>` elements
- Navigation must use `<nav>` with `aria-label`
- Live price data must use `aria-live="polite"` regions
- Sponsored and press release content must include `aria-label="Sponsored content"` on the article wrapper

### Price & Data Accessibility
- Price change indicators must use text prefix (+/–) alongside color, never color alone
- Screen reader text: "Bitcoin price: $94,250, up 4.7 percent in the last 24 hours"
- Heatmap and chart components must have text-based data table alternatives
